Lezione 004 - La CPU MOS 6510¶

Obiettivo: capire cos’è la CPU del Commodore 64 e come esegue le istruzioni.

🎯 Obiettivi¶

  • capire il ruolo della CPU nel C64;
  • conoscere il ciclo fetch-decode-execute;
  • distinguere istruzioni e dati in memoria.

🧠 Introduzione¶

La CPU (Central Processing Unit) è il “cervello” del computer.

Nel Commodore 64 troviamo il:

MOS 6510

È una versione modificata del famoso MOS 6502.

⚙️ Cosa fa la CPU?¶

La CPU esegue solo 3 operazioni fondamentali:

  • leggere memoria
  • scrivere memoria
  • eseguire istruzioni

Tutto il resto è combinazione di queste tre azioni.

📦 La CPU non “capisce” i programmi¶

La CPU NON sa cosa sia:

  • un gioco
  • uno schermo
  • un suono
  • un colore

Per lei esistono solo:

numeri in memoria

📘 Teoria¶

La CPU del Commodore 64 è il cuore del sistema. Riceve istruzioni dalla memoria, le interpreta e modifica lo stato del computer.

🧭 Il ciclo della CPU¶

La CPU lavora in un ciclo continuo:

1. preleva istruzione
2. la decodifica
3. la esegue
4. passa alla successiva

Questo ciclo si chiama:

fetch → decode → execute

🧠 Dove trova le istruzioni?¶

Le istruzioni sono memorizzate in RAM.

La CPU segue un puntatore speciale chiamato:

Program Counter (PC)

Il PC contiene l’indirizzo della prossima istruzione da eseguire.

📌 Il Program Counter¶

Esempio:

PC = $1000

Significa:

👉 “la prossima istruzione si trova a $1000”

Dopo ogni istruzione, il PC avanza automaticamente.

🧮 Le istruzioni sono numeri¶

Esempio:

LDA #$01
STA $D020

In realtà la CPU vede:

A9 01
8D 20 D0

👉 solo numeri

Nota: LDA #value carica il valore immediato nel registro A (accumulatore). STA address scrive il contenuto del registro A nell'indirizzo di memoria specificato. Qui mostriamo questi mnemonici per illustrare la corrispondenza tra codice Assembly e opcode numerici; le istruzioni saranno spiegate in dettaglio nelle lezioni dedicate.

🧠 Cos’è il 6510 davvero?¶

È composto da:

  • ALU (unità aritmetica)
  • registri interni
  • bus dati
  • bus indirizzi
  • logica di controllo

📦 I registri principali¶

Il 6510 ha registri fondamentali:

  • A (accumulatore)
  • X
  • Y
  • SP (stack pointer)
  • PC (program counter)
  • Status register

⚡ Perché è importante?¶

Perché ogni istruzione Assembly:

  • usa questi registri
  • li modifica
  • dipende da loro

🧠 Concetto chiave¶

La CPU NON esegue “comandi”.

Esegue istruzioni numeriche codificate in memoria.

🤖 Come ragiona il 6510¶

Il 6510 non esegue “comandi” in senso umano. Esegue istruzioni numeriche prese dalla memoria, una dopo l’altra, usando il Program Counter per sapere dove andare.

💡 Esempio pratico¶

Se il Program Counter punta a $1000, la CPU legge l'istruzione presente lì e poi passa automaticamente alla successiva.

⚠️ Errori comuni¶

  • pensare che la CPU comprenda il programma come un testo leggibile;
  • confondere Program Counter e indirizzo di memoria generico;
  • dimenticare che i dati e le istruzioni sono entrambi memorizzati in memoria.

📌 Riassunto¶

  • la CPU esegue istruzioni prelevate dalla memoria;
  • il Program Counter indica la prossima istruzione;
  • il ciclo di esecuzione è fetch, decode, execute.

🔜 Preparazione alla lezione successiva¶

Nella prossima lezione studieremo i registri A, X e Y e capiremo come il 6510 conserva e sposta i dati mentre esegue il programma.

🔎 Approfondimento - Dentro il 6510¶

Il MOS 6510 non è un computer generale: è progettato per leggere istruzioni una dopo l'altra usando un Program Counter e registri ridotti.
